With this file you can use the "OKAIDA" theme of Prism without using CDN or importing more files.
It may not be the best practice of the year, but in this way you can easily share your Documentation with an ultra-customizable theme you can build however you want, in just one js file.

File structure

The file is divided in three main sections:

  • The Prism min.js code and a mapped-object of the initial css file for the "OKAIDA" theme
  • A javascript class that sets a css base layer and that you can customize however you want
  • The events that sets the styles and the breakpoints

In the class you will find a constructor with all the basics element i choosed to customize and 4 methods:

  • A methods for setting a style passing a string or an array of elements
  • A methods for setting the breakpoints with a logic you can find here
  • Two methods that initialize the default settings

How to use it

  1. Download the DocPage.js file
  2. Insert it into your project <script src="DocPage.js"></script>
  3. Now you can use the default classes, edit them, create your own, or completly set another logic on the whole file, you decide.

If you need to send just ONE FILE and maybe to a non-developer, then use the min.js version and paste it directly in the script.

setStyle()

As you wil see i created this function with the only intent to pass an already queried element or an array of the same.
Later on, i implemented it to use it for the css structure of the Prism OKAIDA theme.
Next version will be completed. 

                    

                        setStyle = (element, obj) => {
                            if (Array.isArray(element)) { // --- querySelectorAll restituisce array
                                element.forEach(el => {
                                    for (const key in obj) el.style[key] = obj[key]
                                })
                            }
                            else if (typeof element === 'string') { // --- string multiselector
                                if (element.includes(',')) {
                                    const elementQueries = element.split(', ')
                                    elementQueries.forEach(query => {
                                        const elArray = Array.from(document.querySelectorAll(query));
                                        elArray.forEach(el => {
                                            for (const key in obj) el.style[key] = obj[key]
                                        })
                                    })
                                }
                                else { 
                                    const domElArray = Array.from(document.querySelectorAll(element))
                                    domElArray.forEach(el => {
                                        for (const key in obj) el.style[key] = obj[key]
                                    })
                                }
                            }
                            else { // --- querySelector single element
                                for (const key in obj) element.style[key] = obj[key]
                            }
                        }

switchBreakpoints()

This function is far away to be a perfect breakpoint controller, and it has some limitaion in the usage:

  • Beside the initialStyleSettings, if you change style of one element in one breakpoints, you have to notify the breakpoints that doesn't have the same styles, to restyle the desired settings.
  • For now, you still have to specify the next breakpoint the style changes.

I know it's not really useful at the moment but it's really difficult to build a single function that listen to resize, avoid certain triggers and update that styles in the same logic of the CSS Media Queries.

This is gonna be like this for now because i repeat, i builded this class for my small documentations but there will surely be impromentes when i'll take the code again.
